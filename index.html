An implementation of Conway's Game of Life

Game of Life is a zero-player game, played as a mathematical simulation. The game is set in an infinite two-dimensional grid. Each point on the grid is considered as a cell, which can be either alive or dead. A set of simple rules, based on each cell's life status and its number of neighbours, determines the evolution of the game.

When the game is visualised, patterns of remarkable complexity and beauty begin to emerge, and lifelike forms and behaviours can be observed.

Play

Here you can play the Game of Life, on various well-known seeds. The game state evolves to the next generation 5 times every second.

The visualisations have been created using p5.js. The sketches are embedded, with the program being executed in real time in the browser. All seed and p5 sketch data can be found in the GitHub repo.

Select a seed from the menu to begin.

Docs

An optimised implementation of Game of Life, only computing active cells (a cell being considered 'active' only if it is alive or has at least one neighbour).

Improves on solutions which store data on every cell in a two-dimensional array. For example when counting neighbours, in a two-dimensional array the program must iterate through every cell and check its eight neighbours. This program however can iterate through only the live cells, and increment the counts of its neighbours, saving many iterations every generation.

Additionally, this implementation can be played on the infinite grid. By only tracking live cells, no extra memory is needed to track seeds which travel indefinitely (known as Spaceships).

The program is defined using two objects: Cell and Game

Cell

Cell is an object that represents an active cell.

A cell is defined as active if either it is alive or it has at least one neighbour. All other cells are disregarded, as they cannot affect the evolution of the game state.

A Cell has two properties:

Cell.isLive

a Boolean representing if this Cell is currently alive.

Cell.neighbours

an Integer representing how many neighbours this Cell has.

Game

Game is an object that represents the current game state. It contains the following properties and methods:

Game.cells

Contains all the active cells, represented as Cell objects. These are stored in a object with the following data structure for a n * m grid:

              Game.cells = {
  y[0] : { x[0]: Cell, ... , x[n]: Cell},
                      
                       ...

  y[m] : { x[0]: Cell, ... , x[n]: Cell}
}

For example, given the following grid, represented as a 2D array:

            seed = [
        [ 1, 1, 0 ],
        [ 0, 0, 0 ],
        [ 1, 0, 1 ]
       ]

Game.cells will represent this initial state as:

            Game.cells =  {
      0 : { 0: Cell, 1: Cell },
      2 : { 0: Cell, 2: Cell }
}

By using this data structure, the program gains efficiancy through being able to iterate over only active cells, but also can check a specific coordinate in constant time, retaining the advantage of using an Array over an Object.

Game.width

Stores height of current grid, in cell units

Game.height

Stores height of current grid, in cell units

Game.origin

The origin is defined as the top-left corner of the seed grid. This point is designated as [ 0, 0 ].

The Game.origin property stores the co-ordinates of the top-left corner of its grid, relative to the origin.

This becomes important for seeds that travel across the grid (spaceships). Keeping track of the origin ensures no extra memory is needed as the seed travels further across the infinite grid.

Game.evolve()

Evolves the current state and returns the next generation of the Game.

Encapsulates the entire process by chaining the following four methods.

Game.pad()

Adds a perimeter of empty cells around the current grid.

This allows for counting the neighbours' of the Cells on the edge of the grid.

Game.countNeighbours()

Counts the neighbours for all Cells.

By using the data structure outlined in Game.cells, this step can run more efficiently than if a 2D array was stored.

Iterates over all live Cells (in Game.cells) and increments the neighbour counts of each of its neighbours, first adding a dead Cell if there is no Cell to represent a neighbour.

Game.repopulate()

Implements the rules and repopulates the Game accordingly.

The rules have been condensed and are described as follows:

For every active Cell (iterating over Game.cells) it lives to the next generation if it:

- Is alive and has two or three neighbours.

- Is not alive and has three neighbours.

All other active Cells are removed from the Game.

Game.trim()

Removes any empty border rows and columns.

By using the data structure outlined in Game.cells, this step can run more efficiently than if a 2D array was stored.

Game.duplicate()

Returns a duplicate of the Game object.

Allows Game methods to be pure functions, not mutating the input Game.

Game.grid

Getter that returns a 2D Array representation of the Game.

Game.print()

Formats a string representing current state of Game and prints using console.log().

CLI explorer

There is additionally a Command Line Interface tool for exploring the program. This allows for procedural testing of specific scenarios, allowing the user to view the data at every stage of the game.

The CLI is powered by Node.js, and uses the prompts library to build the interface.

Installation

If you do not already have it installed, you will need to install Node.

To check if you have Node installed, open a terminal and enter:

                    node --version

If installed it will return a version number.

Next, clone the github repo to your local machine. Change the current working directory to where you want the cloned directory, and run:

                    $ git clone https://github.com/aabbabar/game-of-life

Then to install required Node modules, run:

                    $ npm install

Finally, to run the CLI tool:

                    $ npm start
